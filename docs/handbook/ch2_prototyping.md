# Chapter 2: Prototyping

## Objective
Convert requirements into UI/UX and system flow.

---

## Prototyping Steps
1. Define user journey
2. Create wireframe structure
3. Map API endpoints
4. Define database entities

---

## Output Required
- UI wireframe (Figma or sketch)
- API flow diagram
- Database schema draft

---

## Rule
- No coding before prototype approval
