# Chapter 1: Requirements Engineering

## Objective
Define clear, structured, and testable requirements before development.

---

## Functional Requirements
- System must support user authentication
- System must support CRUD operations for core modules
- System must expose API for frontend/mobile

---

## Non-Functional Requirements
- High performance (<200ms API response target)
- Scalable architecture (modular design)
- Secure by default (input validation + authorization)

---

## Rules
- No feature starts without written requirement
- Every requirement must map to a feature spec
