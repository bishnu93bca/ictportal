# 🧠 CURSOR CONTEXT MAP (LARAVEL ENTERPRISE SYSTEM)

This file defines the **mental model of the project** for AI.

---

# 🏗️ 1. PROJECT ARCHITECTURE OVERVIEW

This project follows a **Clean Laravel Enterprise Architecture**:

## Layers:

- Presentation Layer → Controllers / Views / React
- Application Layer → Services / Actions
- Domain Layer → Business Rules / Entities
- Infrastructure Layer → DB / External APIs

---

# ⚙️ 2. CORE DESIGN PRINCIPLES

- Controllers must ONLY handle HTTP requests
- Business logic MUST live in Services or Actions
- Database logic MUST be abstracted via Models/Repositories
- No logic inside Blade templates
- API must be versioned (/api/v1)

---

# 🧠 3. BUSINESS LOGIC STRUCTURE

## Services (Main Logic Layer)
app/Services/

Used for:
- Complex business logic
- Multi-step operations
- Data processing workflows

## Actions (Single Responsibility)
app/Actions/

Used for:
- One specific task only
- Reusable operations

## DTOs (Data Transfer Objects)
app/DTOs/

Used for:
- Structured data passing between layers

---

# 🌐 4. FRONTEND ARCHITECTURE

## Blade System
- Server-rendered UI
- Uses components only
- No logic inside views

## React / Vue (If used)
- Located in resources/js/
- Uses API or Inertia
- No business logic in frontend

## Vite
- Handles all asset bundling
- Must be used for production builds

---

# 🗄️ 5. DATABASE DESIGN RULES

- All schema changes via migrations
- Use Eloquent relationships only
- Always prevent N+1 queries
- Index frequently queried columns

---

# 🔐 6. SECURITY MODEL

- FormRequest validation required everywhere
- Policies for authorization
- No raw SQL without bindings
- No sensitive data in API responses
- File uploads strictly validated

---

# ⚡ 7. PERFORMANCE STRATEGY

- Cache heavy queries using Redis
- Queue long-running processes
- Use pagination for large datasets
- Optimize API payload size

---

# 📡 8. API DESIGN STRUCTURE

- Base URL: /api/v1/
- Always use API Resources
- Standard response format:

{
  "success": true,
  "message": "",
  "data": {}
}

---

# 🧪 9. TESTING STRATEGY

- Use Pest preferred
- Feature tests for full flows
- Unit tests for isolated logic
- Use factories for all test data

---

# 🚀 10. EVENT-DRIVEN SYSTEM

- Use Events for decoupling logic
- Use Jobs for async processing

Example Flow:
UserRegistered → Event → Listener → Queue Job

---

# 🧠 11. AI BEHAVIOR EXPECTATION

When Cursor AI reads this project:

- It must prioritize Services over Controllers
- It must never generate insecure code
- It must follow Laravel conventions strictly
- It must assume production-grade environment
